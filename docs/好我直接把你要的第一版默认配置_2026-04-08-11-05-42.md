# 八、V1 开发时的硬规则建议

为了让模型别写飞，建议代码层再做一层校验：

## 生成后校验

* `duration_sec` 是否落在 combo 范围内
* `shot_count` 是否落在范围内
* 是否出现 `hook_text`
* 是否有 `cta_text`
* `shots` 数量是否匹配
* `on_screen_text` 是否过长
* 如果含 `collection_flex`，是否出现 `collection_sheet`
* 如果含 `commerce_scene`，是否提到 use_case
* 如果含 `comment_driver`，最后一镜是否问题式收尾

如果不通过，就自动再生成一次或做轻修正。
