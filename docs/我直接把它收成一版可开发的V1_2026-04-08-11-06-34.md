可以，下面我直接给你一版 **可开发的 V1 方案**。

这版方案的核心不是“固定模板”，而是：

**后台配置视频类型组合 → AI 按组合动态生成 TikTok 分镜脚本**

# 一、V1 目标

## 要解决的问题

你现在已经有：

* 热点抓取
* 贴纸设计生成
* brief 和受众信息

你缺的是把这些信息变成：

**适合 TikTok 品牌号发布的视频分镜脚本**

而且这个脚本不是单一风格，而是允许后台选择不同类型组合，比如：

* 热点共鸣 + 视觉展示
* 热点共鸣 + 商品场景
* 视觉展示 + 整包展示 + 轻转化

## V1 的产出

V1 先做到这件事：

**给定一个贴纸 pack 和一组选中的视频类型，系统自动生成一条可直接交给剪辑使用的 TikTok 分镜脚本。**

## V1 不做

先不做：

* 自动视频渲染
* 自动发布
* 自动评测
* 自动回收平台表现数据
* 复杂 A/B 系统

# 二、产品定义

这不是“视频模板系统”，而是：

## 可配置视频类型组合驱动的脚本生成系统

后台负责：

* 定义视频类型
* 配置类型组合
* 控制时长、镜头数量、语气边界
* 决定这次要测试哪种组合

AI 负责：

* 编排顺序
* 组织镜头
* 写 hook / 字幕 / CTA
* 生成分镜脚本

# 十二、Prompt 设计

不要让 prompt 写得太散。

建议拆成两类。

## 1）Plan Prompt

目标：

* 先决定顺序和意图

它关心：

* 选中了哪些 type
* 每种 type 的目的是什么
* 目标平台是 TikTok 品牌号
* 不要写最终句子太多
* 先输出结构化 plan

## 2）Script Prompt

目标：

* 基于 plan 输出正式分镜

它关心：

* 每个镜头的时长
* 文案短、自然、TikTok 化
* 不要过度广告腔
* 必须有产品 reveal
* 必须有 CTA
* 每屏不宜过长

---

# 十三、V1 约束规则

为了不让模型发散，系统层面要加这些硬约束：

* 视频时长：7–12 秒
* 镜头数量：3–5 个
* 每屏文字尽量不超过 8 个词
* 必须出现至少 1 次 hero sticker
* 如果有 `collection_flex`，必须出现 collection sheet
* 如果有 `commerce_scene`，必须出现 use_case 表达
* 如果有 `soft_sell` 或 `comment_driver`，最后一镜必须含 CTA
* 语气必须符合品牌号：playful / internet-native / lightly sarcastic / not cringe
