# 多渠道上架 · 功能规划与路线图

> 目的:把贴纸包(master 本地产品)**一键铺到 TikTok Shop / Shopify / 亚马逊**,内容尽量 AI 生成、人工最少。
> 本文档列出「我能帮你做什么 → 怎么做 → 依赖与要你拍板的点 → 建议顺序」,供你勾选决定。
> 更新时间:2026-06-24

---

## 0. 现状速览(已经做好的)

| 模块 | 状态 |
|---|---|
| **本地产品库(master)** | 单一主数据源:标题/描述HTML/卖点/关键词/SKU/类目/默认库存/默认价格/AI文案缓存 |
| **TikTok Shop** | 跨店 SKU 统一;批量改库存;每店「▲上架/▼下架」(上架已智能:没上架的自动发布创建,已下架的重新上架);工具栏「更多▾」收纳 |
| **Shopify** | AI 文案生成器(分区富排版 HTML,已缓存+可重生成覆盖);本地**样式预览页**;批量同步(草稿);价格字段;同步跟踪表;「🎨生成全套图片」按钮(复用 master 图集管线) |
| **图片生成** | 已有 `auto_design_images`(master 级):用 Gemini 图生图,产出 封面/总览/生活场景/平铺/使用中 等整套图,写入主版本图集(TikTok+Shopify 共用) |
| **中间层(multi-channel-api, :8000)** | 已具备 TikTok / Shopify / 亚马逊(SP-API Listings)/ FBA 等端点 |

> 关键架构:**主版本图集 + 文案是唯一来源**,各渠道从它取数。改一处、各渠道同步。

## A. Shopify —— 收尾与打磨(最接近闭环,优先级最高)

### A1. 生活场景图 / 全套图片(补齐唯一缺口)
- **是什么**:同类店(Big Moods 等)标配「总览图 + 单张近景 + 生活场景(笔电/水杯/手账)+ 平铺」。你现在多数产品只有 1–2 张。
- **怎么做**:复用已有 `auto_design_images`。Shopify 预览页的「🎨生成全套图片」按钮接好了,但默认 `mode=all` 会重做主图(和 TK 主图冲突)。
  - **改法(已和你定)**:改成**保留已有主图、只补副图**;没主图时才生成主图。→ 不动 TK 主图。
- **依赖**:Gemini 出图(已接);pack 要有贴纸源图。
- **要你拍板**:每个产品生成几张副图(建议 3–4);生活场景选哪些(笔电/水杯/手账/礼盒)。
- **工作量**:小(按钮已在,改个 mode + 加场景提示词)。

### A2. 描述模板增强:阶梯折扣 + 信任区
- **是什么**:从同类店学到的两个转化点 —— ①「买多省多」阶梯折扣话术 ②「30天退换 + 防水可洗实测 + 产地」信任区。
- **怎么做**:加进 `copy_generator.build_body_html` 的固定模板分区(Material&Care / Shipping&Guarantee),折扣话术做成可开关。
- **依赖**:无(纯模板)。折扣**真实生效**需在 Shopify 后台或中间层配活动(另说)。
- **要你拍板**:折扣阶梯数值(如 2件9折/3件8折);是否只写文案 vs 真挂折扣。
- **工作量**:小。

### A3. 端到端真实联调(验证)
- **是什么**:真的把 1 个产品同步成 Shopify 草稿,核对图/文案/价格/SKU/标签是否正确。
- **怎么做**:设价 → 同步 → 打开 Shopify 后台核对 → 修问题。
- **依赖**:中间层 :8000 在跑 + Shopify 凭证(已配)。
- **工作量**:小(但必须做,才能确认管线真的通)。

### A4.(可选)Collection 归类 + 批量同步 UX
- 自动按主题/关键词归类到 Shopify collection;本地产品页批量同步入口已具备,可加筛选「未同步」。
- **工作量**:中。

## B. 亚马逊 —— 新建(价值高、工作量大)

### B1. Listing 上品(STICKER_DECAL)
- **是什么**:把贴纸包上成亚马逊商品。亚马逊字段是**刚性结构**(和 Shopify 自由 HTML 不同)。
- **怎么做**:
  - 新建「亚马逊文案生成器」(AIRouter 两步法),产出亚马逊风格的 `item_name / bullet_point×5 / product_description / search_terms / 属性枚举(color/material/theme…)`。
  - 拼成 SP-API `attributes`,经中间层 `POST /api/v1/amazon/listings` 上品。
  - 复用 master 的 SKU/库存/图片。
- **依赖(关键缺口)**:
  - **图片必须公网 URL**(亚马逊服务端抓图,localhost 不行)→ 需图床/对象存储,或临时公网隧道。**这是硬前提。**
  - 权威必填字段以中间层 `GET /amazon/product-types/STICKER_DECAL` 实时为准。
- **要你拍板**:价格、品牌(是否已亚马逊**品牌备案**)、站点(先美国站?)、发货方式(FBM 自发货 / FBA)。
- **工作量**:大。

### B2. A+ Content(图文详情页)
- **是什么**:亚马逊详情页中部的模块化富图文(横幅/对比表/场景图)。
- **怎么做**:中间层**目前没有 A+ 接口**,需先补 SP-API A+ Content API(建文档→上传图→关联ASIN→提交审核)。AI 出模块文案 + 按尺寸合成模块设计图(接 Gemini)。
- **依赖**:**必须品牌备案**;中间层补接口;A+ 图是"设计图"(带排版的 banner),要专门做。
- **工作量**:很大。**建议最后做。**

## D. 跨渠道 / 运维(打底)

- **缓存策略**(已定):文案存 `shopify_content_json`,只有点「重新生成」才覆盖;不自动失效。
- **同步状态看板**:本地产品页已显示 Shopify 同步态;可加亚马逊态、失败原因、一键重试。
- **成本控制**:出图/AI 调用计数与上限(已有部分);可加每次生成的预估提示。
- **守护**:真实写操作均有二次确认 + 草稿优先,不自动上线。

## 建议顺序(我的推荐)

1. **A1 全套图片(保留主图)** + **A3 端到端联调** → 让 Shopify 真正闭环、出图齐全。 *(最快见效)*
2. **A2 折扣+信任文案** → 提升转化,顺手做。
3. **B1 亚马逊 Listing**(先解决 B 的图片公网 URL 前提 + 你确认价格/品牌/站点/FBM) → 打通第二平台。
4. **C 尺寸参考图 / 总览拼图增强** → 按需补。
5. **B2 亚马逊 A+** → 最后,且需品牌备案 + 中间层补接口。
