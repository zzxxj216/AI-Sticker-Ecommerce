# 规划：多卡包并行生成 + 话题预览 + 按话题对话修改

## 1. 目标与背景

- **保留现有流程**：多轮对话收集信息 → 用户选择话题 → 确认组合/单独生成 → 确认卡包、样式、内容 → 生成图片。该流程不变。
- **新增流程**：面向**批量生成卡包**的工具型流程：
  - **多线程**：一次可生成 **6 个卡包**，并行执行；
  - **规模**：每个卡包 **50–60 张**贴纸；
  - **风格**：每个卡包有**单独风格**，6 个卡包之间风格**互不重复**，卡包内部风格**统一**；**若用户已输入风格等信息，须根据现有信息调整**（见下 §3.3）；
  - **自动确认**：卡包话题组合、样式由系统自动确认（无需每步人工确认）；
  - **话题结构**：一个卡包可包含**多个话题**，自动确定话题数量并**凑齐 50–60 张**；
  - **内容与预览**：先按话题生成各话题内容及每话题下的 image prompt（**每个话题建议 6–8 张**）；再进入**话题预览**步骤：**把每个话题下的多张 prompt 合并，生成该话题的一张「话题总图」**；**每个卡包展示的是这些话题总图的集合**（即每话题一张话题总图，一包多张）。若用户确认没问题再生成每个卡包的具体单张图片——**该「确认后再生成单张」步骤暂不实现**；
  - **对话修改**：用户可通过对话**按「话题 + 某张」定位**到对应图片的 prompt，进行修改并重新生成该张。

## 2. 新流程概览

```
输入：用户提供总主题、风格等（或由系统生成 6 个卡包方向）；**根据用户已有输入调整风格等**
  ↓
【自动】为 6 个卡包分配风格（在用户未指定时互不重复；若用户指定则基于该信息衍生或统一）
  ↓
【自动】每个卡包：确定多个话题、凑齐 50–60 张（每话题 6–8 张）
  ↓
【自动】每个卡包：按话题生成内容 + 每个话题下的 image prompt
  ↓
【话题预览】每个话题：把该话题下的多张 prompt 合并 → 生成该话题的一张**话题总图**；每个卡包展示 = **这些话题总图的集合**（每话题一张，一包多张）；6 包可并行
  ↓
（可选、暂不实现）用户确认话题总图没问题 → 再生成每个卡包的具体单张图片
  ↓
【自动】每个卡包：按 prompt 生成全部单张图片（50–60 张），多卡包并行
  ↓
【对话编辑】用户可说「话题 X 下的第 Y 张」→ 查看/修改该张 prompt → 仅重生成该张
```

## 3. 核心约束与设计

### 3.1 多线程：一次 6 个卡包

- **工具形态**：可为独立脚本/API，或集成进现有 Bot 的「批量模式」。
- **并发**：最多 **6 个卡包** 同时生成（例如 6 个线程/协程，每个卡包一个）。
- **每个卡包**：独立 config（主题、多话题、风格、50–60 张概念），互不共享状态；仅共享 PackGenerator/Claude/Gemini 等服务实例，注意 API 限流与连接池。

### 3.2 每个卡包 50–60 张，多话题凑齐，每话题 6–8 张

- **目标数量**：每包 **50–60 张**（可配置，如 `target_count=55`）。
- **每话题张数**：**每个话题最好 6–8 张**图片，便于该话题下多图合并成一张预览图。
- **话题数量**：一个卡包内可有 **多个话题**，由系统自动决定话题列表；每话题 6–8 张，话题数 ≈ 50÷6～60÷8（约 7–10 个话题）凑齐 50–60。
  - 例：主题「程序员日常」→ 话题「加班」「摸鱼」「开会」「咖啡」「修 bug」「周末」等，每话题 6–8 张，共 50–60。
- **自动确认**：不要求用户逐步确认「话题组合」「样式」；可在工具入口一次性传入或由系统根据总主题自动生成 6 个卡包方向 + 风格。

### 3.3 风格：根据用户输入调整，卡包内一致

- **根据现有信息调整**：若用户已输入**风格、色彩、主题**等信息，必须基于这些信息来设定 6 个卡包，而不是忽略用户输入。例如：
  - 用户指定「统一赛博霓虹风」→ 6 个卡包可共用该风格或在该风格下做 6 种微调；
  - 用户指定「每个包不同风格」→ 在用户偏好范围内分配 6 种互不重复风格；
  - 用户只给主题未给风格 → 再由系统为 6 包分配互不重复的风格。
- **卡包间**：在用户**未**指定统一风格时，6 个卡包的风格**互不重复**（如 6 种：赛博霓虹 / 手绘水彩 / 复古漫画等）；在用户指定后，按上述规则衍生或统一。
- **卡包内**：一个卡包共用一套 **style_guide**（视觉风格、色彩、线条等），该包内所有话题、所有贴纸的 prompt 都遵循同一 style_guide，保证**风格一致**。

### 3.4 话题级内容与 prompt，再按话题生成话题总图，卡包展示话题总图集合

- **按话题生成内容**：每个卡包先按「多话题」拆分，再对每个话题生成：
  - 该话题下的**概念列表**（description + text_overlay），**每话题 6–8 张**；
  - 结合卡包统一 style_guide，生成该话题下每张的 **image_prompt**。
- **每个话题生成一张话题总图**：把**该话题下**的多张图片 prompt 合并，用 `generate_preview_prompt(topic_name, sticker_ideas_该话题, style_guide, ...)` + `generate_preview_image(preview_prompt)` 生成**该话题的一张话题总图**。每个话题一张，不做多话题混在一张图。
- **每个卡包展示话题总图的集合**：一个卡包包含多个话题 → 就有多张话题总图（每话题一张）；**卡包层级展示的是这些话题总图的集合**（多张图：每话题一张话题总图）。6 个卡包可并行，每包输出多张话题总图。
- **确认后再生成单张（暂不实现）**：设计上可在用户确认话题总图没问题后，再生成每个卡包的具体单张图片；**该「确认后再生成」步骤当前不实现**。
- **单张生成**：每个卡包用已生成的全量 ideas（多话题合并列表）按现有 `_generate_images` 生成 50–60 张单图。

### 3.5 按「话题 + 某张」定位并修改 prompt

- **数据结构**：每个卡包的生成结果需带**话题维度**，例如：
  - `pack_id` → 多个 `topic_id` / `topic_name`；
  - 每个话题下：`sticker_index_in_topic`（1-based）或全局 `sticker_index`（1-based） + `image_prompt`、`image_path`。
  - 这样用户可说「话题『加班』下的第 3 张」或「第 2 个话题的第 5 张」。
- **对话解析**：Extractor/Chat 解析出：
  - `topic_identifier`（话题名或序号）；
  - `sticker_index`（该话题内第几张，或全局第几张，依产品约定）；
  - 修改意图或新 prompt 文本。
- **修改与重生成**：根据定位找到对应 Sticker 的当前 `prompt`；若用户给的是意图则用 Claude 转为新 prompt；调用 `regenerate_single_sticker(pack, sticker_index, new_prompt)` 只重生成该张，并返回新图。

## 4. 阶段定义（Phase）

| 阶段 | 常量 | 说明 |
|------|------|------|
| 收集/触发 | `PHASE_COLLECTING` 或专用入口 | 用户或系统提供总主题/方向，触发「6 卡包批量」 |
| 自动规划 | （内部） | 6 卡包风格分配、每包多话题、凑齐 50–60 张 |
| 话题内容+Prompt | （内部） | 每包按话题生成概念与 image prompt |
| 话题预览 | `PHASE_PREVIEW` | 每个话题合并该话题下的 prompt 生成一张**话题总图**；每个卡包展示**话题总图的集合**（每话题一张）；6 包可并行。「确认后再生成单张」暂不实现。 |
| 单张生成 | （内部） | 每包 50–60 张图生成（可并行 6 包） |
| 编辑单张 | `PHASE_EDITING` | 用户按「话题 + 某张」查找并修改 prompt，重生成 |

说明：`PHASE_PREVIEW`、`PHASE_EDITING` 已在 `src/models/generation.py` 中定义。

## 5. 数据与模型扩展

### 5.1 卡包内「话题」结构

- 在 **GenerationConfig** 或生成中间结果中，支持「按话题分组」的 concepts/ideas（**每话题 6–8 张**）：
  - 例如 `topics: List[TopicGroup]`，每个 `TopicGroup`: `name: str`, `concepts: List[StickerConcept]`（6–8 个）, `ideas: List[Dict]`（含 image_prompt）；用于按话题生成预览图与编辑定位。
  - 或保持扁平 `sticker_concepts`，但每个 concept 带 `topic_id` / `topic_name`，以便按话题筛选出 6–8 张做该话题预览与编辑时定位。
- **StickerPack** 或持久化结果中，每个 Sticker 需带 `topic_id` / `topic_name` 与在话题内的序号，便于「话题 X 下第 Y 张」的定位。

### 5.2 会话状态（批量 + 编辑）

- 当前批量任务：`_active_batch: List[GenerationConfig]` 或 `_active_packs: List[StickerPack]`（6 个卡包）。
- 编辑阶段：保存每个 pack 的 `pack_id`、每张的 `topic`、`index`、`prompt`、`image_path`，供对话定位与 `regenerate_single_sticker` 使用。

### 5.3 SessionResponse 扩展

- `generation_stage: Optional[str]`  
  - `"preview"`：本轮执行「6 卡包并行生成预览图」。  
  - `"individual"`：本轮执行「6 卡包并行生成单张」（或仅对已确认预览的卡包）。  
- `ready_for_generation` 与 `configs`：可一次带 6 个 config；预览阶段按话题生成**话题总图**（每话题一张），每包展示话题总图集合；单张阶段调用 `generate_from_config`。

## 6. 后端能力（PackGenerator / 新模块）

### 6.1 已有能力（复用）

- `generate_preview_prompt`, `generate_preview_image`, `generate_preview_from_config`
- `generate_from_config`（单包：style_guide → concepts→ideas → images）
- `_convert_concepts_to_ideas`（可扩展为按 topic 分组输入/输出，便于按话题生成 prompt）

### 6.2 需新增或扩展

- **6 包风格分配**：给定用户输入（主题、风格等）或 6 个卡包方向，**根据现有信息调整**：若用户已指定风格/色彩等，基于该信息衍生或统一 6 包风格；否则为每个包分配一种互不重复的风格（Claude 或预定义风格池）。
- **多话题凑齐 50–60**：给定一个卡包主题，生成多个话题并分配每话题张数，使总和在 50–60（Claude 或规则）。
- **按话题生成内容与 prompt**：对每个话题生成 6–8 个 concepts，再结合包级 style_guide 生成每张的 image_prompt；输出结构按话题分组，便于**按话题**做预览与编辑定位。
- **每话题一张话题总图**：对**每个话题**，把该话题下的 6–8 个 ideas 合并，调用 `generate_preview_prompt(topic_name, sticker_ideas_该话题, style_guide, ...)` + `generate_preview_image` 生成**该话题的一张话题总图**；每个卡包 = 多张话题总图（每话题一张）的**集合**。6 包可并行，每包内多话题可并行或串行。
- **并行执行**：6 包并行；每包内按话题生成话题总图，再 6 包并行单张生成（「确认后再生成单张」步骤暂不实现）。
- **单张重生成**：`regenerate_single_sticker(pack, sticker_index, new_prompt, ...)`（可带 topic 维度：pack 内唯一 index 或 topic + index_in_topic）。
- **按话题定位 Sticker**：根据 `topic_name` / `topic_id` + 序号返回对应 Sticker 的 prompt/path，供对话展示与修改。

## 7. 话题预览步骤：每话题一张话题总图，卡包展示话题总图集合

- **原则**：每个卡包包含**多个话题**，每个话题下有多张图片的 prompt。**把该话题下的多张 prompt 合并**，生成**该话题的一张话题总图**。**每个卡包展示的是这些话题总图的集合**（即每话题一张话题总图，一包多张）。
- **输入**：每个卡包已有 style_guide + 按话题分组的 ideas（每个话题 6–8 个 ideas，含 image_prompt）。
- **步骤**（对**每个话题**分别执行）：
  1. 取该话题下的 sticker_ideas（6–8 条）。
  2. 调用 `generate_preview_prompt(topic_name 或 pack_name + topic_name, sticker_ideas_该话题, style_guide, use_claude=...)` 生成该话题的预览文案。
  3. 调用 `generate_preview_image(preview_prompt)` 得到**该话题的一张话题总图**。
  4. 对同一卡包内其余话题重复 1–3。
- **结果**：每个话题一张话题总图；每个卡包 = 多张话题总图（每话题一张）的**集合**；6 个卡包可并行，每包内多话题可并行或串行。
- **后续**：「用户确认话题总图没问题后再生成每个卡包的具体单张图片」**这一步骤暂不实现**。

## 8. 对话编辑：按话题找图并改 prompt

- 用户输入示例：
  - 「话题『加班』下的第 3 张的 prompt 改成更温馨一点」
  - 「第二个话题的第 5 张不要文字」
- 解析结果：`topic_identifier`（加班 / 第 2 个话题）、`sticker_index_in_topic` 或全局 `sticker_index`、修改内容（新 prompt 或意图）。
- 系统：根据 topic + index 找到对应 Sticker，若为意图则用 Claude 生成新 prompt；调用 `regenerate_single_sticker`；返回新图并继续处于 `PHASE_EDITING`。

## 9. 实现顺序建议

1. **数据与模型**  
   - 为 concepts/ideas 增加 `topic_id` / `topic_name`（或引入 `TopicGroup`）；Sticker 持久化时带上 topic 与在话题内序号。  
   - SessionResponse 增加 `generation_stage`；会话状态支持 6 包 + 编辑阶段缓存。

2. **风格与话题规划**  
   - 实现「6 包风格互不重复」分配（Claude 或规则）。  
   - 实现「单包多话题、凑齐 50–60 张」的规划逻辑（生成话题列表 + 每话题张数）。

3. **按话题生成内容与 prompt**  
   - 扩展或新增：按话题调用 concept 生成 + concepts→image_prompt，输出带 topic 的 ideas 列表；合并后送入现有 preview 与单张生成。

4. **话题预览 + 并行**  
   - 实现**每话题一张话题总图**：对每个话题合并该话题下的 6–8 个 ideas → `generate_preview_prompt` + `generate_preview_image`；每个卡包展示**话题总图的集合**（每话题一张）。6 包可并行，每包内多话题可并行或串行。不实现「用户确认后再生成单张」的等待逻辑。

5. **6 包并行单张生成**  
   - 线程池/协程下对 6 个 config 分别 `generate_from_config`，注意 API 限流与错误隔离。

6. **regenerate_single_sticker + 按话题定位**  
   - 实现单张重生成；实现「话题 + 序号 → Sticker」查询与 prompt 修改。

7. **编辑阶段对话**  
   - `_process_editing_phase()`：解析「话题 X 下第 Y 张」+ 修改意图，调用 Claude 改 prompt（如需）与 `regenerate_single_sticker`，返回新图。

8. **可选**  
   - 将默认/最大 sticker_count 提升到 60；CONCEPT_GENERATOR 支持 50–60 张及多话题说明。

## 10. 与现有流程的兼容

- **现有对话流程**：不启用批量时，仍为「单包、用户逐步确认、可预览可单张」的原有逻辑，不受影响。
- **新批量工具**：通过独立入口或「批量模式」触发 6 包并行、自动确认话题与样式、话题预览与按话题编辑，与现有单包流程并存。
